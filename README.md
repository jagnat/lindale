Audio plugin framework written in Odin. Currently targeting VST3 on Mac and Windows. Work in progress.

My aim is to create a light plugin wrapper to allow for prototyping and exprimenting with audio plugin development, and to have a long-term learning project. The project is built around a hot-reloading architecture, so that the core audio processing and rendering/UI code can be recompiled and reloaded on the fly without restarting the plugin or the DAW.

### Setting up dev environment
- Symlink the `out/Lindale.vst3` folder into your system's VST3 plugin folder (windows: `C:\Program Files\Common Files\VST3`, mac: `~/Library/Audio/Plug-Ins/VST3`)
- Optionally, symlink the `out/hot` folder into your system-dependent SDL3 application preferences folder (windows: `%APPDATA%\user\Lindale`, mac: `~/Library/Application Support/user/Lindale`). This folder will be generated by loading the VST3 if it doesn't exist.
